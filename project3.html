개발 내용 Description

자재 조달 및 물류 관리 부서에서는 공장(Plant)과 보관 위치(Storage Location)별로 실시간 자재 재고 현황을 파악하는 것이 병목 없는 재고 관리를 위해 매우 중요합니다.

이를 돕기 위해 SAP 자재 정보 마스터 및 실시간 재고 원장 테이블인 MARA(일반 자재 데이터), MARC(공장별 자재 데이터), MARD(보관 위치별 자재 데이터)를 고속 조인하고 통합 매핑하는 커스텀 ALV 보고서 화면을 구축하였습니다.

For inventory managers, tracking current stock levels across various plants and storage locations is vital for smooth material flows. I developed a custom ALV report that consolidates material master details and real-time inventory balances from standard SAP tables: MARA (General Material Data), MARC (Plant Data for Material), and MARD (Storage Location Data for Material).

트러블슈팅 & 해결 방안 Troubleshooting & Solution

문제 상황: 초기 구현 후 약 50,000건의 자재 원시 데이터셋을 로드하는 스트레스 테스트 중, 보고서 로딩 시간이 45초 이상 지연되는 심각한 성능 저하 현상이 관찰되었습니다. 원인은 자재 정보 마스터 내부 테이블을 루프(Loop) 도는 과정에서 공장별 정보 데이터 획득을 위해 수행한 중첩 루프 구조로 인해 전체 처리 시간 복잡도가 O(N*M)의 2차 시간 복잡도를 형성했기 때문이었습니다.

해결책: 루프 내부에서의 선형 스캔 방식(Linear Search) 대신, 매핑의 대상이 되는 공장 데이터 내부 테이블(lt_marc)을 정합 키(matnr) 기준으로 오름차순 사전 정렬(SORT)한 뒤 READ TABLE ... BINARY SEARCH 구문을 적용하였습니다. 탐색의 시간 복잡도를 O(N)에서 O(log N)으로 단축함으로써 대량 데이터 매핑 부하를 제거하였고, 결과적으로 리포트 로딩 속도를 기존 45초에서 1.5초 미만으로 단축시켰습니다.

Challenge: During initial runs with production-sized datasets (approx. 50,000 materials), the report took more than 45 seconds to load. The bottleneck was identified as nested loops parsing internal tables to map plant information to material master rows, leading to quadratic time complexity O(N*M).

Solution: Instead of executing standard linear searches within loop structures, I sorted the target internal table (lt_marc) by its key field (matnr) and implemented the READ TABLE ... BINARY SEARCH statement. This reduced the search complexity from O(N) to O(log N), slashing the report rendering time to under 1.5 seconds.

zns_mm_inv_optimization.abap ABAP 
* Sort the target internal table by search key
SORT lt_marc BY matnr.

* Loop through main material list
LOOP AT lt_mara INTO ls_mara.
  CLEAR ls_output.
  ls_output-matnr = ls_mara-matnr.
  ls_output-mtart = ls_mara-mtart.

  * Use Binary Search for high-speed table mapping
  READ TABLE lt_marc INTO ls_marc 
    WITH KEY matnr = ls_mara-matnr
    BINARY SEARCH.

  IF sy-subrc = 0.
    ls_output-werks = ls_marc-werks.
    ls_output-steuc = ls_marc-steuc.
    APPEND ls_output TO lt_output.
  ENDIF;
ENDLOOP.
